JavaScript

Computes key. hashCode() and spreads (XORs) higher bits of hash to lower.
Because the table uses power-of-two masking, sets of hashes that vary only in bits above the current mask will always collide.

because we use trees to handle large sets of collisions in bins, we just XOR some shifted bits in the cheapest possible way to reduce systematic lossage, as well as to incorporate impact of the highest bits that would otherwise never be used in index calculations because of table bounds.
